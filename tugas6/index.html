₊˚⊹♡ 2. Pengantar Generic ♡⊹˚₊

  • 🎀 Apa itu Generic? Fitur untuk membuat kode yang bekerja dengan berbagai tipedata.
  • 🎀 Analog: Kotak penyimpanan yang bisa menampung apa saja, tapi dengan labeltipe.
  • 🎀 Manfaat:
  • i. Type safety (keamanan tipe)
  • ii. code reusability (penggunaan ulang)
  • iii. Compile-time checking

₊˚⊹♡ 3. Masalah Tanpa Generic ♡⊹˚₊

❌ Tanpa Generic (Error)

• Tipe data tidak terkontrol → bisa menyebabkan runtime error.

  dynamic data;  // Bisa apa saja

  Box(this.data);

  var box = Box("Hello"); // isi String
  int value = box.data;   // ❌ dipaksa jadi int

  print(value);
}

✅ Dengan Generic (Sukses)

• Tipe data terjamin → program aman dan berjalan benar.

//Nim: 251410027
//Kelas: SI2A

class Box {

  Box(this.data);
}

void main() {

  int value = box.data;   // ✅ aman

  print(value);
}

₊˚⊹♡ 4. Solusi dengan Generic ♡⊹˚₊

❌ Error

• Karena sudah ditentukan Box, maka hanya boleh menyimpan String. Saat diisi int, error langsung muncul sebelum program dijalankan (compile-time error).

  stringBox.data = 123; // ❌ ERROR
}

✅ Sukses

• Generic () memungkinkan tipe data ditentukan sejak awal. Sehingga Box hanya bisa menyimpan int, dan program menjadi aman (type-safe).

₊˚⊹♡ 5. Sintaks Generic Dasar ♡⊹˚₊

  • 🎀 Type parameter:
  • ◆ → tipe umum
  • ◆ → elemen (biasanya untuk koleksi seperti list)
  • ◆ → key dan value (biasanya untuk map/dictionary)
  • 🎀 Konvensi:
  • ◆ T (Type) → mewakili tipe data secara umum
  • ◆ E (Element) → elemen dalam koleksi
  • ◆ K (Key) → kunci pada map
  • ◆ V (Value) → nilai pada map
  • ◆ R (Return) → tipe nilai yang dikembalikan (return type)

₊˚⊹♡ contoh Sintaks Generic pada class dan function ♡⊹˚₊

Program ini menggunakan generic (``) supaya satu class dan function dapat dipakai untuk berbagai jenis data. `MyClass` berfungsi menyimpan data, `getData()` untuk mengambilnya, dan `myFunc` mengembalikan nilai sesuai tipe. Dengan begitu, kode menjadi lebih fleksibel dan aman.

  T data;

  MyClass(this.data);

  T getData() {
    return data;
  }
}

T myFunc(T value) {
  return value;
}

void main() {
  // Generic class
  var myIntClass = MyClass(10);
  var myStringClass = MyClass('test');

  print(myIntClass.getData());    // Output: 10
  print(myStringClass.getData()); // Output: test

  // Generic function
  var intValue = myFunc(100);
  var stringValue = myFunc('hello');

  print(intValue);    // Output: 100
  print(stringValue); // Output: hello
}

₊˚⊹♡ 11. Pembatasan Tipe dan manfaatnya ♡⊹˚₊

Slide ini menjelaskan tentang pembatasan tipe (type constraints) dalam pemrograman, biasanya dipakai di konsep generic (misalnya di Dart, Java, dll). Berikut penjelasan tiap poin dengan bahasa yang lebih mudah dipahami:

  • • Keamanan Tipe yang Lebih Baik : Memastikan bahwa kode hanya bekerjadengan tipe data yang kompatibel, sehingga mengurangi risiko runtime error.
  • • Peningkatan Keterbacaan Kode : Membuat kode lebih jelas dan mudah dibacakarena kita dapat melihat tipe data yang diharapkan secara eksplisit.
  • • Memungkinkan Penggunaan Method dari Tipe Data Tertentu : Memungkinkankita untuk memanggil method yang spesifik untuk tipe data yang dibatasi.
  • • Mencegah Penggunaan Tipe Data yang Tidak Sesuai : Mencegah penggunaantipe data yang tidak sesuai dengan logika kode.
  • • Membantu dalam Desain dan Implementasi : Membantu kita dalam merancangdan mengimplementasikan kelas dan fungsi generic dengan lebih terstruktur danterdefinisi.

₊˚⊹♡ Pembatasan Tipe pada Interface dan Class ♡⊹˚₊

❌ Error

➤ Jika fungsi dipanggil dengan tipe int, maka akan terjadi error. Hal ini karena int bukan bagian dari Iterable, sehingga tidak bisa digunakan dalam perulangan for. Akibatnya, muncul pesan error yang menyatakan bahwa int bukan turunan dari Iterable. 

//Nama: Aghista Bunga Kirani
//Nim: 251410027

✅ Sukses

  // Output: 1, 2, 3

  printIterable>({'apple', 'banana'}); 
  // Output: apple, banana

₊˚⊹♡ 12: Pembatasan Tipe dengan Class Abstract ♡⊹˚₊

Jika AnimalCage diisi dengan tipe data seperti String, maka program akan menghasilkan error. Hal ini terjadi karena String bukan turunan dari Animal, sehingga tidak memenuhi batasan . Akibatnya, objek tersebut tidak dapat digunakan dalam class AnimalCage dan program gagal dijalankan. .

❌ Error 

//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A

void main() {
  var stringCage = AnimalCage("test");
}

✅ Sukses

➤ Pada kode tersebut, Animal merupakan abstract class yang menjadi dasar bagi Dog dan Cat. Class AnimalCage menggunakan generic dengan batasan , sehingga hanya objek turunan Animal yang dapat digunakan. Ketika AnimalCage diisi dengan Dog atau Cat, program berjalan dengan baik karena keduanya memiliki method makeSound(). Saat method sound() dipanggil, program akan menampilkan suara sesuai jenis hewan, yaitu "Woof!" untuk anjing dan "Meow!" untuk kucing. 

//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A

abstract class Animal {
  void makeSound();
}

class Dog implements Animal {
  @override
  void makeSound() {
    print('Woof!');
  }
}

class Cat implements Animal {
  @override
  void makeSound() {
    print('Meow!');
  }
}

class AnimalCage {
  T animal;
  AnimalCage(this.animal);

  void sound() {
    animal.makeSound();
  }
}

void main() {
  var dogCage = AnimalCage(Dog());
  dogCage.sound(); // Output: Woof!

  var catCage = AnimalCage(Cat());
  catCage.sound(); // Output: Meow!
}

₊˚⊹♡ 13. Praktik 1 – Generic Class & Fungsi ♡⊹˚₊

✦ Generic digunakan agar class dan fungsi bisa dipakai untuk berbagai tipe data tanpa harus membuat ulang kode. Class Repository berfungsi menyimpan dan mengambil data dengan tipe bebas. Fungsi swap digunakan untuk menukar posisi data dalam list. Sedangkan class Triple digunakan untuk menyimpan tiga nilai dengan tipe yang bisa berbeda. Dengan generic, kode menjadi lebih fleksibel, efisien, dan mudah digunakan kembali.

DART SOURCE CODE
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A

class Repository {
  List _data = [];

  void save(T item) {
    _data.add(item);
  }

  T getById(int id) {
    return _data[id];
  }

  List getAll() {
    return _data;
  }
}

void swap(List list, int i, int j) {
  T temp = list[i];
  list[i] = list[j];
  list[j] = temp;
}

class Triple {
  A first;
  B second;
  C third;

  Triple(this.first, this.second, this.third);
}

void main() {
  // contoh penggunaan sederhana
  var repo = Repository();
  repo.save("A");
  repo.save("B");
  print(repo.getAll());

  var list = [1, 2, 3];
  swap(list, 0, 2);
  print(list);

  var triple = Triple(1, "Test", 2.5);
  print("${triple.first}, ${triple.second}, ${triple.third}");
}

₊˚⊹♡ 14. Generic vs Dynamic vs Object ♡⊹˚₊

🔹 Generic

  • • Tipe sudah ditentukan sejak awal (misalnya int, string).
  • • Dicek saat compile-time → lebih aman, error ketahuan lebih cepat.
  • • Tidak perlu casting.
  • • Performa paling bagus.

🔹 dynamic

  • • Tipe tidak dicek saat compile, tapi saat program jalan (runtime).
  • • Sangat fleksibel → bisa isi apa saja.
  • • Tapi rawan error kalau tipe tidak sesuai.
  • • Tidak perlu casting.

🔹 object

  • • Semua tipe bisa masuk karena object adalah induk semua tipe.
  • • Tapi saat dipakai, harus casting balik ke tipe asli.
  • Bisa kena boxing/unboxing (khusus value type seperti int).
  • • Lebih aman dari dynamic, tapi kurang praktis. 
// Contoh perbedaan

// Generic: aman (type-safe)
Box box1 = Box(42);

// Dynamic: tidak aman
dynamic box2 = 42;

// Object: perlu casting
Object box3 = 42;

₊˚⊹♡ 15. Studi Kasus – Cache Generic ♡⊹˚₊

✦ Slide ini menjelaskan penggunaan Generic pada class Cache di Dart. Class Cache digunakan untuk menyimpan data dengan tipe fleksibel (T) menggunakan Map. Method set() untuk menyimpan data, get() untuk mengambil data, dan remove() untuk menghapus data. ` Pada main(), ditunjukkan bahwa cache bisa digunakan untuk berbagai tipe, seperti String (userCache) dan Product (productCache), sehingga lebih fleksibel dan tetap aman (type-safe).

DART SOURCE CODE
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A

class Cache {
  final Map _storage = {};

  void set(String key, T value) {
    _storage[key] = value;
  }

  T? get(String key) {
    return _storage[key];
  }

  void remove(String key) {
    _storage.remove(key);
  }
}

class Product {
  String name;
  int price;

  Product(this.name, this.price);

  @override
  String toString() {
    return 'Product(name: $name, price: $price)';
  }
}

void main() {
  // Cache untuk String
  var userCache = Cache();
  userCache.set('name', 'Aghista');
  print(userCache.get('name'));

  // Cache untuk Product
  var productCache = Cache();
  productCache.set('p1', Product('Laptop', 1000));
  print(productCache.get('p1'));
}

// 1. Class Stack
class Stack {
  final List _items = [];

  void push(T value) {
    _items.add(value);
  }

  T? pop() {
    if (_items.isEmpty) return null;
    return _items.removeLast();
  }

  T? peek() {
    if (_items.isEmpty) return null;
    return _items.last;
  }
}

// 2. Fungsi mergeLists
List mergeLists(List a, List b) {
  return [...a, ...b];
}

// 3. Class Response
class Response {
  T? data;
  String? error;

  Response({this.data, this.error});
}

// Challenge: Generic Repository
class Repository {
  final List _data = [];

  void add(T item) {
    _data.add(item);
  }

  List getAll() {
    return _data;
  }

  void remove(T item) {
    _data.remove(item);
  }
}

// Contoh penggunaan
void main() {
  // Stack
  var stack = Stack();
  stack.push(10);
  stack.push(20);
  print(stack.peek()); // 20
  print(stack.pop());  // 20

  // mergeLists
  var hasil = mergeLists([1, 2], [3, 4]);
  print(hasil); // [1, 2, 3, 4]

  // Response
  var res = Response(data: "Sukses");
  print(res.data);

  // Repository
  var repo = Repository();
  repo.add("Data1");
  repo.add("Data2");
  print(repo.getAll());
}
