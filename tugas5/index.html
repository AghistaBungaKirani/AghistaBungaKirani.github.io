₊˚⊹♡ Polimorfisme & Abstraksi ♡⊹˚₊

  • Polimorfisme: Objek subclass sebagai superclass.
  • Abstract class: tidak bisa diinstansiasi, bisa punya method abstract/concrete.
  • Interface: implicit di Dart, gunakan implements.
  • Contoh cepat: 
 abstract class Bentuk {
    double hitungLuas();
}
  class Persegi implements Bentuk {
    @override double hitungLuas() => sisi * sisi;
}

₊˚⊹♡ Multiple Inheritance dengan Mixin ♡⊹˚₊

  • 💗 Dart menggunakan keyword Mixins untuk menggunakan beberapa class yang berbeda dalam satu class, tanpa perlu membuat inheritance yang rumit.
  • 💗 beberapa bahasa yang mendukung multiple inheritance adalah C++, Python,Scala, Perl, DLL sedangkan Java, C#, dan Kotlin mendukung Single inheritance
  • 💗 Multiple inheritance (pewarisan ganda/banyak) digunakan untukmenyelesaikan masalah di mana sebuah objek (subclass) memiliki perilaku atauproperti dari dua atau lebih induk yang berbeda (superclass) secara bersamaan,menggabungkan fungsionalitas tanpa redundansi kode. Contohnya adalahpemodelan objek kompleks seperti "Bebek" (Mewarisi bisa berenang dan bisaterbang) atau "Smartphone" (Mewarisi Kamera dan Telepon). 3

₊˚⊹♡ Studi Kasus: mixin dengan state ♡⊹˚₊

DART SOURCE CODE
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


// Mixin dengan properti (state)
mixin CanFly {
  bool _isFlying = false; // private

  bool get isFlying => _isFlying;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    } else {
      print('Already on the ground!');
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false; // private

  bool get isSwimming => _isSwimming;

  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    } else {
      print('Already swimming!');
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    } else {
      print('Already not swimming!');
    }
  }
}

// Base class
class Bird {}

// Implementasi class dengan mixin
class Duck extends Bird with CanFly, CanSwim {}

class Fish with CanSwim {}

class Airplane with CanFly {}

void main() {
  // Bird (Duck)
  Duck duck = Duck();
  duck.fly();      // Started flying!
  duck.fly();      // Already flying!
  duck.land();     // Landed!
  duck.land();     // Already on the ground!

  print('Is Duck flying? ${duck.isFlying}');
  print('----------------');

  duck.swim();     // Started swimming!
  duck.swim();     // Already swimming!
  duck.stopSwim(); // Stop swimming!
  duck.stopSwim(); // Already not swimming!

  print('Is Duck swimming? ${duck.isSwimming}');
  print('----------------');

  // Fish
  Fish fish = Fish();
  fish.swim();
  fish.stopSwim();

  print('Is Fish swimming? ${fish.isSwimming}');
  print('----------------');

  // Airplane
  Airplane airplane = Airplane();
  airplane.fly();
  airplane.land();
}

₊˚⊹♡ Pengantar Class Modifiers ♡⊹˚₊

  • 🎀 iii. Desain API yang lebih ekspresif
  • Macam-macam: sealed, base, final, interface (Dart 3.0+)

₊˚⊹♡ Sealed Class ♡⊹˚₊

♡₊˚ 🦢・₊✧Fitur:

  • 🌸 Tidak bisa di-extend atau di-implement dari luar library
  • 🌸 Membatasi hierarki tipe yang diketahui saat compile-time
  • 🌸 Perfect untuk pattern matching (switch exhaustive)
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


// libsealed.dart (bisa juga satu file saja untuk demo)

// sealed class (parent)
sealed class ApiResponse {}

// subclass Success
class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

// subclass Error
class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

// fungsi untuk handle response (pattern matching)
void handleResponse(ApiResponse response) {
  switch (response) {
    case Success(:final message):
      print('Success: $message');
      break;
    case Error(:final errorMessage):
      print('Error: $errorMessage');
      break;
  }
}

void main() {
  ApiResponse res1 = Success("Data berhasil diambil");
  ApiResponse res2 = Error("Gagal mengambil data");

  handleResponse(res1);
  handleResponse(res2);
}

₊˚⊹♡ Keuntungan Sealed Class ♡⊹˚₊

  • Exhaustive checking: Compiler tahu semua kemungkinan subclass
  • Pattern matching: Switch case yang aman
  • Documentation: Jelas hierarki terbatas
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


// Sealed class
sealed class ApiResponse {}

// Subclass
class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

// Function untuk handle response
void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("✅ Success: ${response.message}");
  } else if (response is Error) {
    print("❌ Error: ${response.errorMessage}");
  } else {
    print("Unknown response");
  }
}

void main() {
  ApiResponse res1 = Success("Data berhasil diambil");
  ApiResponse res2 = Error("Terjadi kesalahan");

  handleResponse(res1);
  handleResponse(res2);
}

₊˚⊹♡ Final Class ♡⊹˚₊

Final class adalah class yang tidak bisa di-extend atau diimplementasikan dari luar library, tetapi masih bisa dikembangkan di dalam library yang sama, sehingga berguna untuk menjaga class tetap tertutup dari modifikasi dan lebih aman serta stabil.

//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


final class Settings {
  final String theme;
  final int fontSize;

  Settings({
    required this.theme,
    required this.fontSize,
  });

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

base class CustomSettings extends Settings {
  CustomSettings(String theme, int fontSize)
      : super(theme: theme, fontSize: fontSize);
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();

  CustomSettings customsetting = CustomSettings("Dark", 14);
  customsetting.showSettings();
}

₊˚⊹♡ Perbandingan Class Modifiers ♡⊹˚₊

  • 🎀sealed: Class ini membatasi pewarisan hanya di dalam file yang sama, sehingga subclass tidak bisa dibuat dari luar. Cocok untuk mengontrol struktur inheritance.
  • 🎀base: Class dirancang sebagai dasar untuk diwariskan, terutama dari luar library, tetapi tidak digunakan untuk implementasi langsung seperti interface.
  • 🎀final: Class tidak dapat diwariskan ke luar, namun masih bisa digunakan atau dibuat turunan terbatas di dalam lingkup yang sama.
  • 🎀default: Class tanpa modifier khusus, sehingga tidak memiliki batasan; bisa diwariskan dan digunakan di mana saja.

₊˚⊹♡ Praktik 1 – Sealed Class dengan Pattern Matching ♡⊹˚₊

Sealed Class dengan Pattern Matching adalah konsep di Dart untuk mengatur dan menangani berbagai kondisi secara aman dan terstruktur.

DART SOURCE CODE
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print('Loading...');
      break;

    case Loaded(data: final data):
      print('Data Aghista: $data');
      break;

    case Failed(error: final error):
      print('Error Aghista: $error');
      break;
  }
}

void main() {
  handleStatus(const Loading());
  handleStatus(const Loaded('Data Aghista berhasil diambil'));
  handleStatus(const Failed('Terjadi kesalahan pada data Aghista'));
}

₊˚⊹♡ Studi Kasus: Sistem State Management ♡⊹˚₊

Studi Kasus: Sistem State Management adalah contoh penerapan bagaimana sebuah aplikasi mengatur berbagai kondisi (state) selama berjalan.

DART SOURCE CODE
//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


sealed class AppState {}

class LoadingState extends AppState {}

class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

// Handler
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
      break;

    case SuccessState(items: final items):
      showItems(items);
      break;

    case ErrorState(message: final message):
      showError(message);
      break;
  }
}

void showLoading() {
  print('⏳ Loading...');
}

void showItems(List items) {
  print('✅ Success! Items:');
  for (int i = 0; i < items.length; i++) {
    print('   ${i + 1}. ${items[i]}');
  }
}

void showError(String message) {
  print('❌ Error: $message');
}

void main() {
  print('=' * 60);
  print('DEMONSTRASI SEALED CLASS + PATTERN MATCHING');
  print('=' * 60);

  final states = [
    LoadingState(),
    SuccessState(['Item 1', 'Item 2', 'Item 3']),
    ErrorState('Koneksi gagal'),
    SuccessState(['Dart', 'Flutter', 'Pattern Matching']),
    ErrorState('Data tidak ditemukan'),
  ];

  for (int i = 0; i < states.length; i++) {
    print('\n[${i + 1}] State: ${states[i].runtimeType}');
    print('-' * 60);
    updateUI(states[i]);
  }

  print('\n' + '=' * 60);
  print('KEUNGGULAN SEALED CLASS:');
  print('=' * 60);
  print('✅ Exhaustive switch: semua kasus tertangani');
  print('✅ Type-safe: aman dari error');
  print('✅ Pattern matching: ambil data langsung di switch');
  print('✅ Clear hierarchy: struktur class jelas');
  print('=' * 60);
}

₊˚⊹♡ Ringkasan & Best Practices(Kesimpulan) ♡⊹˚₊

Penggunaan sealed, base, dan final class dalam Dart membantu membuat struktur program lebih aman, terkontrol, dan mudah dipahami. Sealed class cocok untuk menangani kondisi terbatas dengan pattern matching, base class digunakan sebagai dasar yang hanya bisa di-extend, sedangkan final class melindungi class agar tidak bisa diwariskan. Dengan menerapkan ketiganya sesuai kebutuhan, kode menjadi lebih rapi, konsisten, dan minim kesalahan.

₊˚⊹♡ Bonus base Class Repository ♡⊹˚₊

base class Repository adalah class dasar yang digunakan untuk mengelola data dengan konsep CRUD (Create, Read, Update, Delete). Class ini menggunakan generic sehingga bisa dipakai untuk berbagai jenis data.

//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


base class Repository {
  final List _data = [];

  void create(T data) {
    _data.add(data);
    print('Tambah: $data');
  }

  void delete(int i) {
    if (i >= 0 && i < _data.length) {
      print('Hapus: ${_data[i]}');
      _data.removeAt(i);
    }
  }
  void showAll() {
    print('\nData saat ini:');
    _data.asMap().forEach((i, e) => print('${i + 1}. $e'));
  }
}

final class UserRepository extends Repository {}

void main() {
  final repo = UserRepository();

  repo.create('Data 1');
  repo.create('Data 2');
  repo.delete(0);

  repo.showAll();
}

₊˚⊹♡ Bonus Final Class Mathutils (Static Methods) ♡⊹˚₊

`final class MathUtils` adalah class yang tidak bisa diwariskan dan berisi fungsi-fungsi statis. Static method memungkinkan fungsi dipanggil langsung tanpa membuat objek. Class ini digunakan untuk operasi umum seperti factorial, cek bilangan prima, dan pembulatan agar lebih praktis.

//Nama: Aghista Bunga Kirani
//Nim: 251410027
//Kelas: SI2A


final class MathUtils {
  static int add(int a, int b) => a + b;
  static int subtract(int a, int b) => a - b;
  static int multiply(int a, int b) => a * b;
  static double divide(int a, int b) => a / b;

  static void showResult(String label, num value) {
    print('$label = $value');
  }
}

void main() {
  MathUtils.showResult('10 + 5', MathUtils.add(10, 5));
  MathUtils.showResult('10 - 5', MathUtils.subtract(10, 5));
  MathUtils.showResult('10 × 5', MathUtils.multiply(10, 5));
  MathUtils.showResult('10 ÷ 5', MathUtils.divide(10, 5));
}
